
# Codex Agent 迭代调优服务 — 产品需求文档

## 1. 产品概述
- **产品名称**：Agent tune kit
- **目标用户**：开发者
- **使用方式**：通过手工触发的 Codex Skill 扩展，无需 Python 命令行自动执行完整流程
- **目标**：提供一套通用、可复用的服务，用于对不同类型 Agent 进行单轮迭代调优，通过手工执行 Codex Skill 完成测试、异常筛选、归因分析、调优，并生成最终 Markdown 报告。
- **特点**：
  - 门槛低：开发者可直接手动执行 Skill
  - 通用性强：支持多类型 Agent，数据集格式不限
  - 全流程由 Codex Skill 完成，手工控制执行顺序
  - 单轮迭代，多轮迭代可手工触发
  - 输出报告 Markdown 格式，便于阅读和版本管理
  - **数据不确定时提示机制**：模块在处理不确定数据（如字段命名或格式不一致）时，需要引导 Agent 与用户确认，避免误解

---

## 2. 模块设计

### 2.1 输入准备模块（Codex Skill）
- **功能**：
  - 自动解析数据集（CSV/JSON/Excel）
  - 自动识别有效列和字段
  - 收集 Agent 服务代码及配置
  - 整合测试配置和判定规则
  - **数据确认机制**：
    - 当发现字段命名或格式存在不确定性时，Skill 应引导 Agent 与用户确认具体列和数据格式

### 2.2 批量测试脚本生成模块（Codex Skill）
- **功能**：
  - 根据 Agent 类型和数据集生成 Python 测试脚本 `test_runner.py`
  - 包含输入输出记录逻辑和日志管理
  - **数据确认机制**：
    - 对测试数据中存在潜在字段不一致或缺失时，Skill 提示用户确认使用的字段和格式
- **输出**：
  - `test_runner.py`
  - `results_schema.json`
- **依赖处理**：
  - `test_runner.py` 直接使用项目已有的虚拟环境及依赖
  - 避免额外引入新的依赖或生成独立 `requirements.txt`，确保可在现有环境中直接执行

### 2.3 批量执行模块（用户手工执行生成的测试脚本）
- **功能**：
  - 用户手动运行 `test_runner.py`
  - 简单机制实现批量测试执行和结果收集
- **输出**：
  - 所有输出统一为 CSV 文件，包括测试结果和日志
  - 不再生成 `results.json`

### 2.4 异常筛选模块（Codex Skill 或脚本执行）
- **功能**：
  - 两种模式：
    1. **规则模式**：Skill 生成筛选脚本，如果筛选脚本存在则用户直接运行，输出异常数据 CSV 文件
    2. **大模型模式**：通过 Skill 触发大模型进行异常筛选，基于 `results.csv` 将异常数据筛选生成新的 CSV 文件
  - **数据确认机制**：
    - 在筛选过程中，如果输入数据存在未识别或不明确字段，Skill 提示用户确认，以避免误判

### 2.5 异常归因分析与最终报告生成（Codex Skill）
- **功能**：
  - 对异常数据进行归因分析
  - 生成最终 Markdown 报告，包含：
    - 执行摘要
    - 测试结果统计
    - 异常数据清单
    - 归因分析
    - 调优建议
- **输出文件**：
```text
/results/analysis.md
```
- **归因示例**：
```json
{
  "prompt_error": 10,
  "tool_invocation_error": 5,
  "timeout": 2,
  "code_bug": 3
}
```

### 2.6 Agent 调优模块（Codex Skill）
- **功能**：
  - 根据归因报告生成调优方案
  - 调整 Prompt、工具链、工作流或检索策略
  - **数据确认机制**：
    - 在调优操作涉及不确定参数或不一致配置时，Skill 提示用户确认修改内容
- **输出**：
  - 新版本 Agent 文件路径

### 2.7 公共要求 — 封装与通用性
- **要求**：
  - 所有模块均通过手工触发的 Codex Skill 完成，无需外部接口
  - 异常筛选模块可灵活选择规则模式或大模型模式
  - 单轮迭代设计，多轮可手工控制执行顺序
  - Skill 模板和内部逻辑保证服务可复用在不同 Agent 上
  - **数据不确定时提示机制**：
    - 模块在处理数据时若存在不确定性，需要引导 Agent 与用户确认字段和格式

---

## 3. 技术栈
- **Codex Skill**：实现数据解析、测试脚本生成、异常筛选、归因分析和 Agent 调优
- **数据标准化**：所有模块输出统一为 CSV 文件，Markdown 报告
- **可扩展性**：新增 Agent 类型仅需增加 Skill 模板，无需修改外部接口

---

## 4. 使用示例（手工执行）
1. 执行输入准备 Skill，生成标准化数据文件，并确认字段信息
2. 执行批量测试脚本生成 Skill，生成 `test_runner.py`，确认输入字段
3. 用户手动运行 `test_runner.py`，所有输出统一为 CSV 文件，生成测试结果文件
4. 执行异常筛选模块：
   - 规则模式：运行 Skill 生成的筛选脚本，如果存在则直接执行，输出 CSV 文件
   - 大模型模式：通过 Skill 触发大模型，对 `results.csv` 中异常数据生成新的 CSV 文件
5. 执行归因分析 Skill，生成最终 Markdown 报告 `/results/analysis.md`
6. 执行 Agent 调优 Skill，确认调优参数，生成新版本 Agent

---

## 5. 输出文件结构
```text
/results/
├── results.csv  # 测试结果
├── logs.csv     # 日志
├── abnormal_cases.csv  # 异常数据
└── analysis.md  # 本轮最终测试报告
```

---

## 6. 产品交付要求
1. **Codex Skill**：
   - 模块化 Skill 模板，覆盖输入解析、测试脚本生成、异常筛选、归因分析/报告、Agent 调优
   - 异常筛选支持规则模式和大模型模式
   - 内置数据不确定性提示机制，引导用户确认字段和格式
2. **配置文件**：
   - Agent 配置
   - 数据集路径
   - 测试规则与判定规则
3. **可安装部署**：
   - Codex Skill 可注册至 Codex 系统
   - 多 Agent 可通过 Skill 模板和配置快速切换使用

---
## 7. 可扩展方向
- 支持多轮迭代闭环（手工触发或自动化扩展）
- 可视化报告或统计图表
- CI/CD 集成示例（可选）
